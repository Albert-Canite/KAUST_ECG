# 四分类异常预测（仅输出 S/V）的原因与修复建议

## 1. 日志暴露的症状
- 训练日志显示：权重采样器 power=1.0 与逆频率 CE 权重 **同时启用**，S/V 的权重高达 7.49/6.60，而 N 仅 0.31（均值约 4.11）。【F:logs.txt†L9-L16】
- 所有验证/泛化混淆矩阵都只有第 1、2 列（S、V）非零，N/O 被完全忽略；第一批样本的 softmax 概率在 0.05~0.09 区间近乎均匀，但 argmax 始终落在 S 或 V，说明决策面被权重强行偏向稀有类。【F:logs.txt†L17-L200】

## 2. 逻辑层面的根因
1) **“采样 + CE 叠加”过度抵消先验**：采样器已把训练分布重采样为近乎均匀，再叠加 7.5×/6.6× 的 CE 权重，相当于把 S/V 的有效损失提升了几十倍，把真实先验彻底翻转，模型最优解就是“无论输入都预测 S/V”。
2) **权重范围未随采样器调整**：权重计算仍按均匀先验、`max_ratio=20`，没有在“已启用采样器”时下调，导致翻倍效应无人制动。
3) **输出层缺乏校准**：从样本概率可以看出 logits 差距并不大，但类别决策被极端权重主导，说明问题在损失配置而非特征表示或数据质量。

## 3. 建议的修改（按风险从低到高）
1) **关闭叠加（首选）**：训练命令加入 `--no-stack-sampler-with-ce`，只保留采样器或只保留 CE 权重，让两者其一负责均衡，避免过补偿。
2) **必须叠加时，减小权重幅度**：
   - 把 `class_weight_power` 降到 0.3~0.5，或把 `class_weight_max_ratio` 降到 4~5；同时在 `train.py` 中，当 `sampler` 不为空时强制使用均匀 CE 权重（覆盖现有可选分支），确保采样器主导再平衡。
   - 增加打印：记录最终 CE 权重和采样器状态，防止出现权重>5、采样器已等频仍继续放大。
3) **权重热身改为“真热身”**：日志显示 warmup 仍是 `alpha=1.0`（等于未生效）。将 `--weight_warmup_epochs` 设为 5~10，并在 `constraints.py` 中把默认 warmup alpha 起点改为 0（线性从 0→1），首轮只用均匀权重，避免一开始就被极端权重锁死在 S/V。
4) **验证步骤**：重新训练时先在前 1~2 epoch 观察混淆矩阵是否开始出现 N/O 召回；若仍全为 S/V，再进一步调低 CE 权重或直接关闭叠加。上述操作应能在不改模型结构的情况下解除预测坍塌。

## 4. 复现与验证要点
- 重新训练时记录权重打印，确认 N/S/V/O 的 CE 权重与采样策略是否符合预期（稀有类>1，但不至于反向碾压常见类）。
- 关注前 1~2 个 epoch 的混淆矩阵是否开始出现 N/O 召回；若仍全为 S/V，再降低 CE 权重或直接关闭叠加。
- 若关闭叠加后仍崩塌，再检查数据管线（类别计数）或模型结构，但当前日志已指向损失配置为主要原因。

## 追加反思（KD 暂停后学生模型仍弱）

### 发现的问题
- 最近日志（Val MacroF1≈0.41、Gen MacroF1≈0.36）显示学生模型在 S/V/O 上普遍低召回，Gen 的 S 类几乎为 0.01，说明训练批次仍被 N 类主导，稀有类曝光不足。
- 早停依据只看 Val MacroF1，但验证集来源于同一批训练记录，患者分布与 Generalization 集差异大，且 MacroF1 中 N 类仍占 25% 的权重，允许“全预测 N”在轻微召回稀有类时触发早停。

### 修正措施
- **默认开启 weighted sampler**：将 `use_weighted_sampler` 默认值设为 `True`，在不叠加 CE 权重的前提下让批次近似均衡，避免稀有类训练信号稀薄。
- **关注稀有类的早停指标**：新增 `abnormal_macro_f1`（对 S/V/O 取均值），早停/保存最好模型使用 MacroF1 与 abnormal MacroF1 的平均值，防止“全 N”模型因 N 类高精度被误判为最佳。
- **降低 CE 权重幂次**：将 `class_weight_power` 默认值下调为 0.5，避免未来手动关闭采样器时重新引入过强的类权重。

### 预期效果与验证
- 训练日志将打印 `(abn …)` 指标，若早期 epoch 中 abnormal MacroF1 持续上升，说明稀有类已被有效抽样参与训练；若依旧停滞，可继续调高采样器 `power` 或加长耐心。新的早停分数会偏向同时提升 S/V/O 表现的模型，缓解 Val/Gen 分布差异带来的“伪最佳”问题。

## 最新实验复盘（准确率≈0.36 停滞）

- 现象：最新日志（Val MacroF1≈0.28/Acc≈0.37，Gen MacroF1≈0.29）显示模型大量把 N 判为 S，混淆矩阵的 S 列远高于其余列，说明采样仍过度推高了少数类先验，导致决策面偏向 S 类。
- 原因：我们默认启用了 weighted sampler，且 `sampler_power=1.0`、无权重上限，等价于把极少数的 S 样本重复抽取到接近平衡甚至更高的比例；当 CE 权重又保持均匀时，模型会把 S 当作“新的多数类”，从而整体倾向 S。
- 修正：
  1) 将采样幂次下调为 `sampler_power=0.7`，
  2) 为采样权重增加 `sampler_max_ratio`（默认 6.0）并记录到日志，防止单类权重爆炸。
  新增打印每类采样权重，训练开始即能确认采样力度。
- 后续验证要点：
  - 观察前 1–2 epoch 的混淆矩阵，确认 N 的召回不再掉到 0.35 以下；如果仍偏向 S，则进一步降低 `sampler_max_ratio`（如 4–5）或暂时关闭采样仅用 CE 权重。
  - 若采样收敛但仍低召回，可考虑重新启用温和的 CE 权重（`class_weight_power` 0.3–0.5）或加长耐心，避免早停过早。

## 进一步复盘（预测坍塌到 N/O）

- 现象：最新日志显示模型几乎只输出 N/O，S/V 完全 0，但早停仍触发，因为 `abnormal_macro_f1` 把 O 也计入，O 的高召回掩盖了 S/V 的 0 分，综合分仍然达标。
- 根因：
  1) 早停指标把 S/V/O 平均，O 较多且易学，导致“全 O”仍拿到可观的 abnormal F1。
  2) 验证分与 N/O 相关的 `macro_f1` 权重过高，未单独凸显 S/V。
- 修正：
  - 新增 `rare_macro_f1`（仅平均 S/V），并将早停分改为 `0.2*macro + 0.3*abnormal + 0.5*rare`，强制 S/V 表现达标才算最佳模型。
  - 训练日志与 JSONL 记录中打印 `rare` 指标，方便发现再次坍塌的迹象。
- 后续验证要点：
  - 重新训练后，若 `rare` 一直停留在 0，说明数据采样或标签覆盖有问题；若 `rare` 有缓慢上升而 `abn` 较高，则表示 O 不再掩盖 S/V。
  - 若 `rare` 波动但总体上升，可以适当降低采样器 `max_ratio` 或小幅增加 CE 权重，以平衡 S/V/O 之间的先验。

## 最新修正（重新引入温和 CE 权重并暴露采样分布）

- 现象：在加入 `rare_macro_f1` 后，模型仍会坍塌到 N/O，表明采样器 + 均匀 CE 权重仍不足以驱动 S/V 学习。
- 直接原因：采样器权重虽然设置了上限，但未显示实际批次占比，也把 CE 权重完全置为均匀，导致 S/V 梯度仍偏弱。
- 代码调整：
  1) **采样器诊断**：`make_weighted_sampler` 现在返回按 “权重×数量” 计算的期望抽样占比，并在训练启动时打印 `Expected batch mix`，可直接看到 N/S/V/O 在批次中的比例。
  2) **温和 CE 权重回归**：即使开启采样器且未显式叠加，也对 CE 权重取平方根（≈温和的逆频率），确保 S/V 梯度不会完全回落到均匀权重；若显式叠加，则使用完整逆频率权重。
- 下一步验证：
  - 观察 `Expected batch mix` 是否接近均衡（或至少 S/V >10%）；若仍过低，可提高 `sampler_power` 或上限。
  - 若 `rare_macro_f1` 仍为 0，说明 S/V 仍未被学习，可考虑开启 `--stack-sampler-with-ce` 或提高 CE 幂次（保持温和），同时检查采样占比是否已达到预期。

## 新一轮改动（聚焦 S/V 暴露度与难例损失）

- 现象：最新日志显示模型依旧无法识别 S 类（F1=0），V 类也偏低；说明采样后的梯度仍不足以驱动稀有类学习。
- 直接原因：
  1) 采样器只做幂次均衡，无法直接将批次配比拉向想要的占比；
  2) 纯 CE（即便带权重）对极度不平衡+硬样本的场景仍不敏感，容易被简单样本主导。
- 代码调整：
  1) **目标化采样配比**：`make_weighted_sampler` 支持 `target_mix`（默认 0.5/0.2/0.2/0.1 对应 N/S/V/O），按“目标占比 / 观测占比”直接计算采样权重，再统一归一化并套上 `max_ratio`。训练启动时打印目标与实际期望占比，方便发现 S/V 曝光不足。
  2) **切换为 Focal Loss**：新增 `--loss_type {ce,focal}`（默认 focal）与 `--focal_gamma`，将 warmup 后的类权重归一化为 `alpha`，配合 focal 的 `(1-p)^gamma` 抑制易分类样本，集中放大 S/V 这类难例梯度。
- 预期验证：
  - 训练首轮即打印目标配比与采样权重，确认 S/V 期望占比≈20% 后再观察 `rare_macro_f1` 是否>0；
  - 若仍 0，可适当提高 `sampler_max_ratio`（如 8–10）或调高 focal `gamma`（2→2.5），再观察是否开始召回 S 类；
  - 若出现 S 召回但 N/O 掉得太多，可把 `target_sampler_mix` 改为略偏向 N/O（如 0.55/0.18/0.18/0.09）或降低 `gamma`。

## 最新反思与修正（防止早停指标压制 O 类）

- 现象：在引入目标采样与 focal 后，模型出现两极：有时全偏向 N/O，有时全偏向 N/V，说明早停分仍在奖励“牺牲某一类”以换取 S/V 或 O 的得分。
- 根因：早停分仍以 MacroF1 + abnormal + rare 的组合为主，`rare_macro_f1` 只看 S/V，导致把 O 误判为 V 也能得到高分，从而压制 O 类。
- 修正：
  1) 监控并打印最小单类 F1（`min_f1`），让任何类别坍塌都能被即时看到。
  2) 将早停评分调整为 `0.4*macro + 0.3*abnormal + 0.3*min_f1`，只要有一类掉成 0，分数就会显著下降，避免“全 V”或“全 O”方案获胜。
- 验证要点：训练早期观察 `min` 字段是否>0；若仍为 0，说明至少有一类未被学到，应先检查采样占比是否覆盖到该类，再考虑调整目标配比或降低 focal `gamma`。

## 追加定位（宏平均未覆盖缺失类别导致早停误判）

- 现象：最新日志出现“只预测 N/V 或 N/O，但早停仍触发，macro 仍有 0.30+”。这是因为宏平均默认只统计出现过的类别，漏掉了被完全忽略的 S/O 类。
- 根因：`compute_multiclass_metrics` 调用 `f1_score` 时未指定标签集合，导致缺失类别不计入分母，分数虚高，早停与最佳模型选择偏向坍塌解。
- 修正：强制使用 `labels=range(num_classes)` 计算精确率/召回/F1 和宏平均，使缺失类别得到 0 分，直接拉低 `macro_f1` 与组合分，阻止漏类模型获胜。
- 验证要点：重新训练时，若某类完全缺失，`macro_f1` 应明显下降且 `min_f1`=0；若依旧高企，再检查采样/权重设置或数据覆盖问题。

## 追加修正（去除默认强平衡并阻止零样本训练）

- 现象：在引入目标化采样与 focal 之后，模型仍在不同类之间来回坍塌；最新日志甚至只剩两类被预测，说明采样/损失组合仍然过度干预。
- 根因：默认启用“目标配比 + focal”的强平衡策略，使梯度分布随类权重轻微变动而剧烈摆动；同时若训练划分偶然缺少某类，还会在零样本下进行训练，必然无法学习该类。
- 修正：
  1) **默认回归温和策略**：关闭加权采样的默认开关、移除目标配比默认值、将损失默认改回 CE，并把采样幂次降到 0.5，避免无监督地强行平衡导致梯度震荡。
  2) **零样本保护**：加载数据后检查训练折是否包含四类，若有缺失直接报错提示重新划分/调整记录，避免在“某类完全缺席”的前提下训练并造成必然坍塌。
- 下一步：在此温和基线下观察四类是否均有预测；如仍有漏类，可再手动开启加权采样或 focal，但需逐步调参并关注新打印的缺类异常。

## 最新分析（062–063 轮次仍崩溃为 N/O 或 N/V）

- 现象：最新日志（Val MacroF1≈0.46、rare=0，Gen MacroF1≈0.31、rare=0）显示训练后仍只学到 N 与 O（或 N 与 V），S/V/O 中至少两类长期为 0。
  训练损失已降到 ~1.06，但 rare/min F1 恒为 0，说明早期根本没有有效梯度覆盖这些类。
- 根因：
  1) **要么“过平衡”，要么“完全不平衡”**：此前为避免过强采样/权重，把默认采样器与目标配比关闭后，训练又回到原始长尾分布，S/V 在早期 epoch
  完全得不到曝光，导致后续梯度永远缺失；而一旦重新开采样又容易过冲。
  2) **训练初期缺乏受控的“先露面”阶段**：没有一个温和且有限的阶段来确保 S/V 至少被看见几轮，导致训练曲线在两极间震荡。
- 修正策略：
  - 引入**温和课程序列**：默认在前 6 个 epoch 使用上限受控的轻量采样（sqrt 逆频率，上限 4×），并配合平方根 CE 权重，只保证 S/V/O 早期出现，避免
    过度平衡；随后切换回自然分布。这样既不会把先验彻底翻转，也能避免“零曝光”导致的永久性坍塌。
  - 日志将提示课程阶段和预计批次占比，方便确认 S/V 在前几个 epoch 有被抽到；若仍未出现在混淆矩阵，可再提高课程序列时长或上限。
